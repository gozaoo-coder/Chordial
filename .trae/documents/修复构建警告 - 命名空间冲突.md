## 问题分析

构建时出现两个警告：
1. `getAlbumArt` 从 `musicSource/index.js` 和 `album.js` 重复导出
2. `getArtistImage` 从 `musicSource/index.js` 和 `artist.js` 重复导出

## 原因

`src/api/tauri.js` 中：
```javascript
export * from './musicSource/index.js';  // 包含 getAlbumArt, getArtistImage
export * from './artist.js';             // 包含 getArtistImage
export * from './album.js';              // 包含 getAlbumArt
```

## 修复方案

修改 `src/api/tauri.js`，显式导入需要的函数，避免使用通配符导出导致的冲突。

## 修改内容

将 `src/api/tauri.js` 中的通配符导出改为显式导出：
- 从 `musicSource/index.js` 显式导入需要的函数
- 或者从 `artist.js` 和 `album.js` 导入时排除冲突的函数

## 文件变更

**文件**: `src/api/tauri.js`
- 移除 `export * from './musicSource/index.js';` 中的冲突导出
- 显式列出需要导出的函数
- 保留 `artist.js` 和 `album.js` 的导出（返回类实例的版本更推荐）
