## 性能问题修复方案

### 问题 1: AMLLyrics ResizeObserver 频繁触发
**文件**: `src/components/player/AMLLyrics.vue`
**修复措施**:
- 添加防抖机制，避免频繁触发初始化
- 只在尺寸从 0 变为有效值时触发初始化
- 移除不必要的 console.log 减少开销

### 问题 2: 强制同步布局
**文件**: `src/components/player/AMLLyrics.vue`
**修复措施**:
- `calcLayout(true)` 改为 `calcLayout()`，避免强制同步布局
- 使用 `requestAnimationFrame` 批量处理布局更新

### 问题 3: 歌词数据重复解析
**文件**: `src/components/player/AMLLyrics.vue`
**修复措施**:
- 对 `lyricLinesData` computed 属性添加缓存机制
- 使用 `shallowRef` 减少响应式开销
- 歌词解析结果缓存，避免重复解析相同内容

### 问题 4: PlayerAlbumCard CSS 优化
**文件**: `src/components/player/PlayerAlbumCard.vue`
**修复措施**:
- 将 `vh`/`vmin` 单位改为更稳定的单位，减少布局抖动
- 优化动画性能，添加 `will-change` 提示

### 问题 5: PlayerBackground 优化
**文件**: `src/components/player/PlayerBackground.vue`
**修复措施**:
- 降低动画复杂度
- 优化渲染参数

### 执行顺序:
1. 先修复 AMLLyrics.vue (影响最大)
2. 再优化 PlayerAlbumCard.vue
3. 最后调整 PlayerBackground.vue

每个修改后都会验证功能正常性。
