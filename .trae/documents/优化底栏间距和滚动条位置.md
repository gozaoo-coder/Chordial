## 问题分析

从截图中可以看到两个问题：

1. **底栏间距问题**：底栏（AppBottomNav）和播放器控制栏（PlayerControlBar）之间有多余的间距
2. **滚动条位置问题**：滚动条显示在了播放器控制栏的右侧，而不是在主内容区域内

## 修复方案

### 1. 修改 AppLayout.vue

**修复底栏间距：**
- 移动端（max-width: 1023px）时，当播放器显示，主内容区域的 `padding-bottom` 需要同时考虑底栏高度和播放器高度
- 将 `padding-bottom: calc(var(--bottom-nav-height, 60px) + 8px)` 改为 `padding-bottom: calc(var(--bottom-nav-height, 60px) + var(--player-bar-height, 100px))`

**修复滚动条位置：**
- 将 `.main-content` 的 `overflow-y: auto` 移到 `.content-wrapper`，使滚动条只在内容区域内显示
- 或者调整 `.main-content` 的高度计算，确保滚动条不会延伸到播放器区域

### 2. 修改 AppBottomNav.vue

- 确保底栏在播放器显示时正确上移，与播放器控制栏紧密贴合

### 具体修改

**文件：src/components/layout/AppLayout.vue**

```css
/* 修改前 */
@media (max-width: 1023px) {
  .main-content {
    margin-left: 0;
    padding-bottom: calc(var(--bottom-nav-height, 60px) + 8px);
  }
}

/* 修改后 */
@media (max-width: 1023px) {
  .main-content {
    margin-left: 0;
    /* 需要同时考虑底栏和播放器的高度 */
    padding-bottom: calc(var(--bottom-nav-height, 60px) + var(--player-bar-height, 100px));
  }
}
```

**文件：src/components/layout/AppBottomNav.vue**

底栏已经有 `with-player` 类来处理上移，需要确保样式正确。

## 预期效果

1. 底栏会紧贴播放器控制栏上方，没有多余间距
2. 滚动条只在主内容区域内显示，不会延伸到播放器控制栏区域
