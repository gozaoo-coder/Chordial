## 问题分析

从控制台输出可以看到：
- `plainLyrics` 包含有效的 LRC 格式歌词（如 `[00:00.00]作词 ： 雷十一`）
- `hasPlainLyrics: true`
- 但页面上歌词区域空白

**根本原因：**
在 `src/utils/lyricConverter.js` 的 `parseLRC` 函数中，用于跳过元数据标签的正则表达式 `/^\[\w+:[^\]]+\]$/` 存在缺陷：

```javascript
// 跳过元数据标签 [ti:], [ar:], [al:], [offset:]等
if (/^\[\w+:[^\]]+\]$/.test(trimmed)) {
  continue;
}
```

`\w+` 会匹配单词字符（包括数字），导致 `[00:00.00]` 这样的时间戳被错误地识别为元数据标签而跳过。

虽然 `[00:00.00]作词 ： 雷十一` 不会匹配（因为后面有歌词文本），但如果歌词中有纯时间戳行（如 `[00:00.00]`），或者解析逻辑有其他问题，会导致歌词解析失败。

**修复方案：**

修改正则表达式，要求元数据标签的键必须以字母开头（而不是数字），这样可以正确区分元数据标签和时间戳：

```javascript
// 修改前
if (/^\[\w+:[^\]]+\]$/.test(trimmed)) {

// 修改后  
if (/^\[[a-zA-Z]+:[^\]]+\]$/.test(trimmed)) {
```

这样 `[ti:标题]`、`[ar:艺术家]` 等元数据标签会被跳过，而 `[00:00.00]` 这样的时间戳不会被误跳过。
